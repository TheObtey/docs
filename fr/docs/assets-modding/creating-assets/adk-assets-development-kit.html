Kit de Développement des Assets

nanos world Assets Development Kit - ADK

nanos world provides a fully configurable and ready Unreal Project called Assets Development Kit (ADK) to help you integrating and exporting Assets and Maps much faster. Ce projet contient des Espaces réservés, Outils et Références correctes qui doit être utilisé pour créer des Caractères , Models, etc.

Note

To be able to use the ADK, you will need to have Unreal Engine installed.

Téléchargement du kit de développement des assets de nanos world - ADK​

Our ADK is uploaded in GitHub, please download it latest version and extract it in your computer.

Ouverture du projet du kit de développement d'assets​

Vous y trouverez un fichier NanosWorldADK.uproject, il suffit de l'ouvrir avec Unreal Engine et vous êtes prêt !

You will notice we have two folders in the root directory: NanosWorld/ and MyAssetPack/:

tip

You can delete or rename MyAssetPack to anything you want to be your Asset Pack.

caution

Veuillez ne pas MODIFIER, SUPPRIMER, CRÉER ou COPIER n'importe quel fichier présent dans le dossier NanosWorld/ sinon, l'ADK ne fonctionnera pas correctement. Veillez également à ne pas les modifier involontairement ! Si l'on vous demande d'y enregistrer un fichier modifié, annulez-le ! Toutes les références doivent utiliser NanosWorld/ là où elles se trouvent.

Le dossier NanosWorld/ contient quelques assets que vous pouvez utiliser pour référencer vos assets et faire en sorte que l'exportation conserve les bonnes références lors de leur utilisation dans le jeu.

Outils disponibles dans le projet ADK​

  • Blueprints placeholders NanosWorld/Blueprints/Placeholders/
  • Générateur de code Lua NanosWorld/Blueprints/Utility/WBP_LuaCodeGenerator
  • Générateur Assets.toml NanosWorld/Blueprints/Utility/WBP_AssetsTomlGenerator
  • Acteur Soleil & Ciel NanosWorld/Blueprints/World/BP_SunSky
  • Matériaux physiques NanosWorld/MaterialLibrary/PhysicalMaterials/
  • Squelette du mannequin NanosWorld/Characters/Common/SKEL_Mannequin
  • Générateur de vignettes NanosWorld/Blueprints/Utility/ThumbnailGenerator

Blueprints placeholders​

NanosWorld/Blueprints/Placeholders/​

Vous pouvez utiliser les Blueprints placeholders pour définir les positions de spawn des véhicules, armes, personnages et props. Cela ne fonctionne que pour vous aider à générer un code Lua avec des positions de spawn après tout. Ces blueprints ne sont pas exportés avec la carte. Use the Lua Code Generator to generate the Lua script with all spawn locations.

Générateur de code Lua​

NanosWorld/Blueprints/Utility/WBP_LuaCodeGenerator​

Lua Code Generator is an Editor Utility Widget which scans the map which is currently loaded in your Unreal and generate a list of spawn code with exact Location and Rotation for every Placeholder Blueprint you had spawned.

Pour l'ouvrir, faites un clic droit dessus et appuyez sur Run Editor Utility Widget :

Générateur Assets.toml​

NanosWorld/Blueprints/Utility/WBP_AssetsTomlGenerator​

This is an Editor Utility Widget which scans a folder (determined by the TextBox) and generates the configuration for an Assets.toml with all Assets in the folder (which can be used in nanos world).

Acteur de Soleil & Ciel​

NanosWorld/Blueprints/World/BP_SunSky​

Il s'agit d'un Sun & Sky Blueprint préconfiguré qui peut être utilisé et facilement remplacé par le soleil officiel de nanos world avec World,SpawnDefaultSun(). Ceci est utile pour obtenir un résultat approximatif de la façon dont la lumière se présentera dans le jeu si vous décidez de faire apparaître le soleil par défaut.

info

Note: Remplacer le Soleil par le Scripting avec World.SpawnDefaultSun(), respawnera l'acteur du Soleil, ce qui signifie qu'aucune configuration faite sur l'acteur Soleil & ; Ciel ne persistera

Exemple : L'intensité lumineuse, la couleur, le post-traitement et toute autre configuration seront perdus.

Matériaux physiques​

NanosWorld/MaterialLibrary/PhysicalMaterials/​

Si vous créez un Matériau, vous pouvez (et devriez) utiliser ces Matériaux physiques pour mieux vous intégrer au nanos world. L'utilisation de ces matériaux physiques permettra à nanos world de reconnaître le type de surface de votre prop/objet et de lui attribuer un son quand touché.

caution

NE PAS modifier ou renommer les Matériaux physiques, mais seulement les utiliser en les référençant dans vos Matériaux.

Squelette du mannequin​

NanosWorld/Characters/Common/SKEL_Mannequin​

To be able to create Skeletal (Character) Meshes which work with our Character class, you must set your Skeletal Mesh to use our Mannequin Skeleton. This Skeleton is the same as Unreal Engine 5 default Mannequin, so any Skeletal Mesh which uses UE4’s Mannequin Skeleton will work for us!

Pour changer le squelette d'un mesh : cliquez dessus avec le bouton droit de la souris >> ; Skeleton >> Assign Skeleton et changez-le en notre SKEL_Mannequin.

Générateur de vignettes​

NanosWorld/Blueprints/Utility/ThumbnailGenerator​

Nous avons lancé notre outil Générateur de vignettes pour vous aider à générer des images .jpg de vos assets. Pour l'utiliser, il suffit d'ouvrir le niveau ThumbnailGenerator et de cliquer sur Play. Vous pourrez ensuite définir un dossier pour la recherche des assets (actuellement, seuls les static et skeletal meshes sont pris en charge) et un dossier pour la sauvegarde des images (en .jpg). Vous pouvez éventuellement conserver la hiérarchie des répertoires dans le dossier de sortie ou enregistrer toutes les images dans le même dossier. Une fois que vous avez appuyé sur Générer, le processus commence, l'outil charge tous les assets et prends une photo à partir de ceux-ci.

info

Note: cela peut être un processus lent selon si vos assets ont déjà été compilés précédemment.

Il est également recommandé de générer deux fois les vignettes, afin de garantir qu'elles seront enregistrées dans la meilleure qualité possible.
