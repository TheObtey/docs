Static Meshes

Un mesh statique est l'asset le plus simple à importer et à utiliser dans nanos world. Comme il n'a pas de squelette, son utilisation est très simple. In nanos world Static Meshes are mainly used for Props and StaticMesh entities.

tip

Veuillez ne pas confondre avec nanos world la classe Static Mesh !

info

Veuillez vous référer à meshes statiques d'Unreal définition et utilisation officielles, également pour des informations sur la façon d'importer un .FBX dans Unreal, comment l'optimiser et ainsi de suite. There are also thousands of tutorials regarding importing a .FBX from Blender (or other Modeling Tool) into Unreal Engine.

Créer et exporter un Static Mesh​

caution

Attention! Cette page est ancienne et ce tutoriel peut être dépassé ! Veuillez vous référer à ADK pour une approche moderne.

caution

Please before continuing, make sure you have and are using the ADK - Assets Development Kit.

It is very simple to create and export Static Meshes, let's create simple Cube imported from a .fbx (Static Mesh) and a Material (applied to the cube) for our Assets. Nous les avons également créés et placés dans un dossier appelé MyPack qui nous aidera par la suite.

Cube Static Mesh et Materiau appliqué au Cube​

Double clique sur le cube puis: Open the static mesh overview​

For exporting them in a recognizable way by nanos world, you need to “Package the Project” (i.e. cooking and packaging it), for that just select the following option and select any folder in your computer and wait it finishes, it may take some minutes:

Après avoir terminé, vous obtiendrez un dossier comme celui-ci :

Importation et utilisation d’Assets sur votre serveur​

Après avoir exporté votre projet, nous copierons manuellement le dossier exporté de celui-ci, celui que nous recherchons sera probablement situé à assets-development-kit/Content/. Comme nous avons créé un dossier appelé MyPack, nos assets exportés seront à assets-development-kit/Content/MyPack/ :

Et c’est tout ! Vous devez juste copier le dossier MyPack/ dans le dossier serveur Assets/ et créer un fichier de configuration appelé Assets.toml dans le dossier MyPack/ avec le modèle décrit dans Assets Pack Configuration File. Pour chaque asset de votre pack, vous devez ajouter une entrée dans le fichier de configuration.

Pour notre mesh Cube, il nous suffit d'ajouter une entrée avec le chemin du cube relatif à notre dossier Assets avec le modèle [ASSET_NAME] = "[ASSET_PATH]" :

Assets.toml
# Static Meshes
[assets.static_meshes]
SM_Cube = "SM_Cube"

Ensuite, il est possible de faire apparaître notre cube comme cela :

Server/Index.lua
Prop(Vector(0, 0, 0), Rotator(0, 0, 0), "MyPack::SM_Cube")
